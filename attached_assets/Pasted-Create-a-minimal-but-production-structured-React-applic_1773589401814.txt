Create a minimal but production-structured React application using Next.js (App Router) and TypeScript.

The application is a simple Video Streaming Platform UI that communicates with a backend API. The frontend must be clean, minimal, and designed for easy backend integration.

Tech requirements:
- Next.js 14+
- TypeScript
- Tailwind CSS
- hls.js for video streaming
- Axios for API requests
- React Query (TanStack Query) for API state management
- JWT authentication support
- Environment variable based API base URL
- Clean modular folder architecture

Design style:
Minimal, modern, dark theme, inspired by Netflix or Vimeo but much simpler.

Pages required:

1. Login Page
Route: /login

Features:
- Email + password login form
- Form validation
- Submit to POST /auth/login
- Store JWT token in httpOnly cookie or localStorage
- Redirect to dashboard after login
- Show loading and error states

2. Dashboard Page
Route: /dashboard

Features:
- Fetch videos from GET /videos
- Display video cards in grid
- Each card shows:
  - thumbnail
  - title
  - duration
- Clicking card navigates to /watch/{videoId}

3. Watch Page
Route: /watch/[id]

Features:
- Video player component
- Uses hls.js to play HLS stream
- Loads manifest from:
  GET /videos/{id}/manifest

Video player should support:
- play / pause
- progress bar
- volume control
- fullscreen
- loading indicator
- buffering indicator

4. Upload Page
Route: /upload

Features:
- Video upload form
- Title field
- Video file input
- Upload progress bar
- POST /videos/upload

Components to generate:

VideoPlayer

apiClient.ts

GET /videos

POST /videos/upload

Auth system:

- Store JWT token
- Attach token to API requests automatically
- Redirect to login if token missing
- Protected routes for dashboard/watch/upload

load manifest URL

load manifest directly.
